File Formats


Galaxy-tools need files with certain file-formats. Wherever in GALAXY you can choose files from your history GALAXY recognises the formats and lets you only choose files with the right file-format.
The most important file-formats used are bam, bed and gtf/gff.

» You can find information on biological file formats here: File Formats
» Some datatypes used on GALAXY are explained here: GALAXY Datatypes
   You will also find descriptions of file formats used on GALAXY below the tool GALAXY STANDARD / Get Data / Upload File from your computer

How to get your data to GALAXY


You have 2 possibilities:

1. Pushing a file to GALAXY from solitaire / forskalle
  1. Log in to GALAXY in case your are not already logged in: GALAXY
  2. Go to your QUEUE and log in and find your file of interest: QUEUE
  3. Go to column called "togalaxy" in your queue (yellow)
  4. Push a file shown under "togalaxy" to GALAXY by clicking on the button of the file you want to push (e.g. green):


  5. The file you pushed will appear in a new history of your GALAXY-account.
    In case you want to work in another history you can copy your pushed files by going to the gear wheel icon
    at the very top right corner of the history-panel of GALAXY and choose "Copy Datasets" from the drop-down-menu:



    Now you can copy any file(s) from the chosen Source History to the chosen Destination History:


or:

2. Uploading a file from your computer

Use the GALAXY-tool GALAXY STANDARD / Get Data / Upload File from your computer. This file should have less than 2GB of memory otherwise it will not be uploaded!!!
You will also find descriptions of file formats used on GALAXY below this tool.

 

GALAXY-TOOLS & WORKFLOWS



Usually you can use the default settings of the tools installed on GALAXY.
But it is recommended to read carefully what the parameters do and change them if necessary or more appropriate for your data!
If you don't know what the parameters do exactly please don't hesitate to come to us and ask. This will provide us the opportunity to improve the explanations in the tools.

Sometimes the results of the tools are split into some files in different formats. This allows to use some of these files in different other tools for further analysis.

- bamCompare normalizes and compares two BAM files to obtain the ratio, log2ratio or difference between them



NGS: deeptools / bamCompare
calculates the coverage for a sample over its input/control

If you don't have an input/control file use:
NGS: deeptools / bamCoverage

- treated sample in bam-format
- control (input) sample in bam-format

bin size (default: 50) - a mean is being calculated for each bin, the higher the value the smoother will be the curve, the higher the value the more precise will it be

How to comapre the two files:
Recommended >
"Compute log2 of the number of reads ratio"

- bigwig (binary format) - recommended to use for the following tool (computeMatrix)
